# 機能実装: 設定画面

## 要件

### 機能要件

#### 設定画面機能
1. **アカウント設定**
   - プロフィール編集
   - メールアドレス変更
   - パスワード変更

2. **プラン設定**
   - 現在のプラン表示
   - プラン変更（プレミアムへのアップグレード）

3. **通知設定**
   - 学習リマインダー
   - 復習通知
   - メール通知設定

4. **言語設定**
   - インターフェース言語の変更

5. **その他**
   - ヘルプ・サポート
   - 利用規約
   - プライバシーポリシー
   - ログアウト

### 非機能要件

- **パフォーマンス**:
  - 設定保存: 500ms以内
  - 画面表示: 500ms以内

- **セキュリティ**:
  - ユーザー認証必須
  - パスワード変更時の確認

## 実装指示

### Step 1: テスト設計

1. **ユニットテスト（Vitest）**
   - コンポーネントのレンダリング
   - 設定変更ロジック
   - バリデーション

2. **統合テスト（Vitest）**
   - 設定保存フロー
   - API呼び出し

3. **E2Eテスト（Playwright）**
   - 設定画面の表示
   - 設定変更の保存
   - ログアウト

テストファイルは `frontend/web/app/settings/page.test.tsx` および `frontend/web/e2e/settings.spec.ts` に配置。

### Step 2: 実装

#### 実装ファイル構造

```
frontend/web/
├── app/
│   └── settings/
│       ├── page.tsx                 # 設定画面
│       └── page.test.tsx            # Vitestテスト
├── components/
│   └── settings/
│       ├── AccountSettings.tsx      # アカウント設定
│       ├── PlanSettings.tsx         # プラン設定
│       ├── NotificationSettings.tsx # 通知設定
│       └── *.test.tsx               # Vitestテスト
└── e2e/
    └── settings.spec.ts             # Playwrightテスト
```

#### APIエンドポイント

```
GET  /api/v1/settings
PUT  /api/v1/settings/profile
PUT  /api/v1/settings/password
PUT  /api/v1/settings/notifications
```

## 制約事項

- 既存の `frontend/web/app/` は変更しない（新規追加のみ）
- `frontend/web/app/settings/` 配下のみ編集可能
